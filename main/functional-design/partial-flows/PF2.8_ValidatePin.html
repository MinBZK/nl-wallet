Partial Flow 2.8 Validate entered PIN

Overview

Aspect

Description

Summary

The system validates a PIN entered by the user. Too many failed attempts causes the app to be temporarily or permanently blocked. This behaviour is configurable.
For example, there may be four rounds of four attempts each. After the first three rounds, the app is blocked temporarily for 1 minute, then 5 minutes, then 1 hour.
After the final round, the app is blocked permanently.

Goal

Preventing unauthorized access to and actions in the app.

Preconditions

None

Postconditions

Depends on the use case.

Used by

  • UC2.3 Unlock the app
  • PF2.4 Confirm a protected action

Parameters

None

Possible Results

  • Valid PIN
  • Retry
  • Back

Additional Documentation

  • PIN Validation

Possible errors

  • No internet
  • Server unreachable

Logical test cases

  • LTC57 PIN is invalid timeout
  • LTC58 PIN is invalid Block
  • LTC59 Mixed PIN entry success and failure
  • LTC60 Timeout reset after successful PIN entry
