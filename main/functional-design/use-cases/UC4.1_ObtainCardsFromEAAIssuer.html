Use Case 4.1 Obtain one or more cards from a (Q)EAA Issuer

Overview

Aspect

Description

Summary

The user adds one or more cards to their NL Wallet and/or updates one or more cards, by starting issuance flow on the issuer’s website,
approving the data request by the issuer using PIN, and then confirming issuance of the offered data with PIN.

Goal

Obtaining and/or updating cards from a (Q)EAA Issuer.

Preconditions

  • The App holds the requested data.

Postconditions

  • Card(s) is/are issued to and stored in the App.
  • The disclosure event is added in the history.
  • The issuance event or card renewed is added in the history

Triggered by

  • User invokes a universal link that contains the URL to a (Q)EAA Issuer.
  • User scan a QR UC9.9 Scan QR

Additional Documentation

  • PIN Validation
  • Issuance
  • Disclosure Based Issuance

Possible errors

  • No internet
  • Server unreachable
  • Session expired
  • Attributes missing
  • Known issuer error
  • Unknown issuer error

Logical test cases

  • LTC2 Issuance fails
  • LTC4 User rejects issued attributes
  • LTC5 Disclosure based Issuance happy flow
  • LTC8 User rejects disclosure of attributes
  • LTC9 User continues after initially cancelling disclosure
  • LTC10 User continues after initially cancelling issuance
  • LTC11 cross-device generic issuance
  • LTC12 Renew card happy flow
  • LTC79 User has no cards to be issued
