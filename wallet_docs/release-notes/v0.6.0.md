# v0.6.0 (dev)

## Upgrading

## New features

- The Wallet Provider will now block recovery codes during PIN recovery or PID
  renewal when the newly disclosed recovery code does not match with stored
  recovery code. This is suspicious because it is already checked by the NL
  Wallet app as well.
- When a card is deleted, the wallet now stores this in its history.
- Allow incoming BLE connection through 'Present QR' screen. The BLE server is
now started when this screen is displayed and a remote verifier can connect to
trigger navigation to the disclosure flow. Actual close proximity disclosure
is not yet implemented.
